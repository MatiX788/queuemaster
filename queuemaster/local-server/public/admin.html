⬇️ APK hier runterladen 🌐 Download-Seite öffnen
⚠️ APK noch nicht verfügbar

Die Tablet-App muss erst gebaut werden. Mit GitHub Actions passiert das automatisch in der Cloud — du musst nichts installieren.

🚀 Schritt-für-Schritt:
  1. GitHub-Account erstellen (kostenlos): github.com/signup
  2. Neues Repository anlegen (z.B. "queuemaster")
  3. Das komplette Projekt hochladen (Ordner per Drag & Drop in GitHub)
  4. GitHub baut automatisch die APK — warte 3-5 Minuten
  5. Unter Actions oder Releases → APK herunterladen
  6. APK hier ablegen: local-server/public/downloads/QueueMaster.apk
💡 Ausführliche Anleitung: siehe APK_BAUEN.md im Projektordner
📱 So installiert der Mitarbeiter die App
  1. Am Tablet im Chrome-Browser die Download-URL öffnen
  2. „APK herunterladen" antippen
  3. Bei der Warnung „Unbekannte Apps zulassen" → Erlauben
  4. „Installieren" antippen
  5. App öffnen → Server-IP eingeben (z.B. http://192.168.1.100:3000)
  6. Fertig — läuft als richtige App im Vollbild

☁️ Cloud-Server (Fernsteuerung der Werbung)
🌐 Wie funktioniert die Fernsteuerung?
Der lokale Server baut beim Start automatisch eine zweite Verbindung (Socket) zum Cloud-Server auf. Über diesen Kanal schickt der Cloud-Admin Werbeanzeigen direkt auf alle Displays — ohne dass jemand vor Ort sein muss. Bestelldaten bleiben immer lokal und funktionieren auch ohne Internet.
